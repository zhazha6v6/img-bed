# 管理后台-录题中心 PRD
所属端：管理后台 H5（PC 为主）　　涉及角色：教师、教研员、平台教研员、系统管理员

> 说明：
> 1. 审核中心为独立 owner；录题中心只负责创建、保存草稿、提交审核前动作和结果文案，不展开审核中心页面细节。

## 一、业务价值与核心流程

**用户痛点：**
当前录题场景既要支持个人沉淀题目，也要支持校本题、平台题共建，还要兼容现有 demo 已有的批量导入、单题录入、资源库同步试卷能力。若不统一在录题中心入口页切题目归属类型，用户会分不清“我是录个人题，还是录校本题/平台题”，研发也难以统一提交结果口径。

**核心主流程：**

1. **进入录题中心：** 用户从一级菜单 `录题中心` 进入，默认看到 `个人题 / 校本题 / 平台题` 切换。
2. **选择题目归属：** 用户先确定当前录入的是 `个人题 / 校本题 / 平台题`，系统在后续入口页、单题录入页、草稿列表中持续带上该上下文。
3. **选择录入方式：** 在当前题目归属下，用户进入 `批量导入`、`单题录入`，或直接查看 `草稿列表`；也可从 `资源库同步试卷` 创建 OCR 任务。
4. **完成录入与保存：** 用户可通过单题录入保存草稿，也可通过批量导入/OCR 形成待处理任务；草稿统一回到 `草稿列表` 继续编辑。
5. **提交审核后继续留痕：** 提交校本题或平台题时，题目仍保留在 `草稿列表`，但状态改为 `待审核`，便于创建人持续查看处理进度。
6. **形成最终结果：** 创建个人题时，题目直接进入个人题库并 `正式生效`；校本题或平台题审核通过后进入对应题库，并从 `草稿列表` 移除；若审核驳回，则题目回到 `草稿中`，原负责人可继续编辑后再次提交；平台题提交前需二次确认。
## 二、用户体验旅程

- **【起点/入口】：** 用户从一级菜单 `录题中心` 进入，默认落在录题中心入口页，页面顶部先切换 `个人题 / 校本题 / 平台题`。
- **【浏览/筛选】：** 用户选择题目归属后，浏览当前归属下的四个入口：`批量导入`、`单题录入`、`查看草稿列表`、`从资源库同步试卷`。
- **【关键交互】：** 用户进入单题录入页手工录题，或进入批量导入页发起 OCR 识别；过程中可保存草稿；已保存的单题草稿和 OCR 任务统一在 `草稿列表` 继续承接。校本题和平台题提交后，草稿列表中的对应记录状态改为 `待审核`。
- **【终点/出口】：** 个人题创建成功后，页面提示“创建成功，已进入个人题库”；校本题和平台题提交成功后，页面提示“提交成功，已进入审核中心待处理”，同时该记录继续保留在 `草稿列表`；审核通过后从草稿列表移除，审核驳回后恢复为可编辑草稿。
- **【异常出口】：** 若字段缺失、OCR 失败、平台题确认取消或提交异常，页面停留当前页，提示具体原因，并保留已填写内容。

## 三、核心数据字典

- **业务实体 A（题目草稿 QuestionDraft）**
  - 草稿 ID（唯一标识）
  - 题目归属类型（枚举：个人题 / 校本题 / 平台题）
  - 录入方式（枚举：单题录入 / 批量导入 / OCR 识别）
  - 草稿状态（枚举：草稿中 / 待审核 / 已驳回 ）
  - 题目基础信息（学科、学段、题型、难度）
  - 题目内容（Markdown 正文、答案、解析）
  - 知识点（可多条）
  - 负责人（当前编辑人）
  - 最近保存时间（时间戳）
  - 提交时间（校本题 / 平台题提交后写入）
  - 审核结果说明（审核通过为空，审核驳回时记录驳回原因）

- **业务实体 B（题目 Question）**
  - 题目 ID（唯一标识）
  - 题目归属层级（枚举：个人 / 校本 / 平台）
  - 题目状态（枚举：正式生效 / 审核中）
  - 题目内容（题干、选项、答案、解析）
  - 学科与学段（业务归类）
  - 绑定知识点（可多条）
  - 创建人（账号）
  - 生效时间（个人题创建成功时间，或审核中心回写通过时间）

- **业务实体 C（OCR任务 OcrTask）**
  - OCR 任务 ID（唯一标识）
  - 任务状态（枚举：等待处理 / 处理中 / 已完成 / 处理失败）
  - 上传文件名
  - 文件大小
  - 输入模式（枚举：题目答案同时输入 / 题目答案分开输入）
  - 页码范围
  - 识别进度（百分比）
  - 识别结果题目数

  **实体关系：**
  录题中心入口页只负责切题目归属与选择录入方式；单题录入和批量导入会生成题目草稿或 OCR 任务；题目草稿补全后可直接形成题目。个人题提交后直接生效；校本题和平台题提交后进入 `待审核`，并继续留在草稿列表中；审核通过后进入对应题库并移出草稿列表，审核驳回后恢复为可编辑草稿。

## 五、交互动作与业务流转

- **动作 1：切换题目归属类型**
  - **前置规则（限制条件）：** 用户已进入录题中心入口页。
  - **业务处理（系统要干什么）：** 系统切换当前上下文为 `个人题 / 校本题 / 平台题`，并同步刷新入口页说明文案、单题录入入口链接、草稿列表入口链接。
  - **成功体验（下一步去哪）：** 用户继续点击批量导入、单题录入、草稿列表、资源库同步试卷中的任一入口，进入对应页面。
  - **异常体验（出错了怎么办）：** 不涉及；当前 demo 为纯前端切换，不依赖服务端请求。

- **动作 2：保存单题草稿**
  - **前置规则（限制条件）：** 用户已进入单题录入页，且已填写部分题目内容。
  - **业务处理（系统要干什么）：** 系统按当前题目归属保存单题草稿，记录学科、题型、知识点、正文、负责人、保存时间等信息。
  - **成功体验（下一步去哪）：** 提示“草稿保存成功”，跳转 `草稿列表`，并在当前题目归属下展示该草稿。
  - **异常体验（出错了怎么办）：** 若字段校验失败或保存异常，停留当前页，提示原因，已填写内容保留。

- **动作 3：创建个人题**
  - **前置规则（限制条件）：** 当前题目归属为 `个人题`；题目必填字段完整。
  - **业务处理（系统要干什么）：** 系统先保存当前草稿，再将状态改为 `正式生效`，并写入个人题库。
  - **成功体验（下一步去哪）：** 页面提示“创建成功，已进入个人题库”；当前题目不再出现在草稿列表中。
  - **异常体验（出错了怎么办）：** 停留当前页，提示缺失字段或创建失败原因，已填写内容保留。

- **动作 4：提交校本题或平台题审核**
  - **前置规则（限制条件）：** 当前题目归属为 `校本题` 或 `平台题`；题目必填字段完整；平台题提交前需二次确认。
  - **业务处理（系统要干什么）：** 系统先保存当前内容，再将草稿状态改为 `待审核`；平台题在提交前弹出确认框，用户确认后才继续提交；提交后的记录继续保留在 `草稿列表` 中。
  - **成功体验（下一步去哪）：** 页面提示“提交成功，已进入审核中心待处理”；用户返回 `草稿列表` 时，可继续看到该记录，状态为 `待审核`。
  - **异常体验（出错了怎么办）：** 若字段缺失、用户取消平台题确认或提交异常，停留当前页并提示原因。

- **动作 5：审核中心回写结果**
  - **前置规则（限制条件）：** 校本题或平台题已提交审核，且审核中心已产出处理结果。
  - **业务处理（系统要干什么）：** 若审核通过，系统将题目写入对应题库，并把该记录从 `草稿列表` 中移除；若审核驳回，系统将草稿状态改回 `草稿中`，并写入驳回原因，保留原负责人。
  - **成功体验（下一步去哪）：** 审核通过后，创建人不再在草稿列表中看到该记录；审核驳回后，创建人可在草稿列表中看到该草稿恢复为可编辑状态，并继续修改后再次提交。
  - **异常体验（出错了怎么办）：** 若审核中心结果同步异常，系统保留当前状态并提示稍后刷新，不得直接丢失草稿记录。

- **动作 6：批量导入 / 从资源库同步试卷**
  - **前置规则（限制条件）：** 用户已选择文件，或已在资源库选中试卷。
  - **业务处理（系统要干什么）：** 系统创建 OCR 任务，写入任务列表，并在草稿列表中展示任务状态；识别完成后可进入后续处理。
  - **成功体验（下一步去哪）：** 用户在草稿列表看到 OCR 任务；任务完成后可点击“处理结果”继续流转。
  - **异常体验（出错了怎么办）：** 若 OCR 失败或处理失败，任务状态展示为 `处理失败`，并保留失败原因。

- **动作 7：在草稿列表继续编辑单题草稿**
  - **前置规则（限制条件）：** 当前题目归属下存在状态为 `草稿中` 或 `已驳回` 的单题草稿。
  - **业务处理（系统要干什么）：** 系统带着 `draftId` 和 `questionOwnership` 打开单题录入页，并回填上一次保存的内容。
  - **成功体验（下一步去哪）：** 用户继续编辑后可再次保存草稿、创建个人题或提交审核；若草稿来自驳回记录，页面应同步展示驳回原因。
  - **异常体验（出错了怎么办）：** 若草稿不存在或状态不允许继续编辑，拦截并提示“草稿不存在或已失效”或“当前草稿不可继续编辑”。

## 六、边界情况与防坑

- **权限隔离：** 当前 demo 默认展示 `个人题 / 校本题 / 平台题` 三种类型切换，未接真实权限裁剪；正式接入时应按角色隐藏无权限题目归属。
- **数据时效：** 个人题创建后直接从草稿列表移出；校本题/平台题提交后状态改为 `待审核` 并继续展示；只有审核通过后才从草稿列表移除；审核驳回后恢复为 `草稿中`。
- **空状态：** 草稿列表无数据时，必须给出明确文案，并提供 `新建导入`、`单题录入` 入口；不可出现空白页。
- **加载状态：** 单题录入提交、保存草稿、图片 OCR、任务刷新都要有明确 Loading 态；OCR 处理中不可误展示为已完成。
- **网络异常：** 保存草稿、提交审核、创建 OCR 任务失败时，停留当前页并保留已录入内容；允许用户重试。
- **高危操作保护：** 平台题提交前必须二次确认；若用户取消确认，不得继续提交。
- **审核结果回写：** 审核中心回写结果时，必须保证状态流转单向清晰：`草稿中 -> 待审核 -> 通过移除` 或 `草稿中 -> 待审核 -> 已驳回 -> 草稿中`，不可出现草稿直接丢失。
- **并发冲突：** 当前 demo 未接入多人协同编辑锁；正式实现时应补“同一草稿被他人更新后需刷新重试”的冲突提示，且驳回后仍应保持原负责人继续编辑。
